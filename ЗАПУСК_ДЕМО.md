## 📋 Режимы запуска

### Режим 1: Docker Full ⭐ Рекомендуется

Frontend и PostgreSQL в Docker, Backend на хосте.

```bash
# Запуск
./start.sh docker-full

# Или просто (режим по умолчанию)
./start.sh
```

**Полезные команды:**
```bash
./start.sh --stop          # Остановить сервисы
./start.sh --logs          # Просмотр логов
./start.sh --clean         # Очистить данные БД
./start.sh --backend       # Запустить только backend
```

---

### Режим 2: SQLite (для тестирования)

Минимальные зависимости - только Rust/Cargo.

```bash
# Первый запуск с инициализацией
./start.sh sqlite --init

# Запуск сервера
./start.sh sqlite
```

**Доступ:** http://localhost:3000

**Учётные данные:**
- `admin` / `admin123`

**Полезные команды:**
```bash
./start.sh sqlite --stop   # Остановить backend
./start.sh sqlite --build  # Пересобрать backend
./start.sh sqlite --logs   # Просмотр логов
```

---

### Режим 3: Docker All (продакшен)

Всё в Docker: Frontend + PostgreSQL + Backend.

```bash
# Запуск
./start.sh docker-all
```

**Доступ:** http://localhost

**Полезные команды:**
```bash
./start.sh docker-all --stop   # Остановить сервисы
./start.sh docker-all --logs   # Просмотр логов
./start.sh docker-all --clean  # Очистить volumes
```

## 🐛 Решение проблем

### Проблема: "Address already in use"

**Решение:**
```bash
# Остановить процессы на порту 3000
pkill -f "semaphore server"

# Или через lsof
lsof -ti:3000 | xargs kill -9

# Запустить снова
./start.sh
```

### Проблема: "Docker не найден"

**Решение:**
```bash
# Установите Docker
# Для Ubuntu/Debian:
sudo apt-get update
sudo apt-get install docker.io docker-compose

# Для macOS: установите Docker Desktop
# Для Windows: установите Docker Desktop
```

### Проблема: Backend не подключается к БД

**Решение:**
```bash
# Проверьте PostgreSQL (для Docker режимов)
docker logs semaphore-db

# Перезапустите БД
./start.sh --restart

# Проверьте .env файл
cat .env | grep SEMAPHORE_DB
```

### Проблема: Frontend не загружается

**Решение:**
```bash
# Проверьте наличие frontend файлов
ls -la web/public/

# Соберите frontend
./web/build.sh

# Перезапустите
./start.sh --clean
./start.sh
```

### Проблема: "Permission denied"

**Решение:**
```bash
# Сделайте скрипты исполняемыми
chmod +x start.sh
chmod +x web/build.sh
```

## 💡 Советы

1. **Используйте режим Docker Full** для разработки - лучший баланс между удобством и производительностью

4. **Сохраняйте токены** - сессия действует 24 часа
5. **Проверяйте логи** при ошибках: `./start.sh --logs`
