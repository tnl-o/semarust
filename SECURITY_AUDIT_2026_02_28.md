## 🔒 Безопасность зависимостей

### Rust зависимости (cargo-audit + osv-scanner)

#### 🔴 Критические уязвимости (1)

| Уязвимость | Серьёзность | Пакет | Решение |
|------------|-------------|-------|---------|
| **RUSTSEC-2023-0071** (Marvin Attack) | 5.9 (Medium) | `rsa 0.9.10` | Нет доступного исправления |

**Детали**:
- Атака Marvin - атака по времени на RSA decryption
- Влияние: **Низкое** - RSA используется только для генерации ключей
- Митигация: Не использовать RSA decryption в критичных по времени операциях

#### 🟡 Предупреждения (2)

| Уязвимость | Пакет | Статус |
|------------|-------|---------|
| **RUSTSEC-2025-0057** | `fxhash 0.2.1` | Не поддерживается |
| **RUSTSEC-2024-0384** | `instant 0.1.13` | Не поддерживается |

**Зависимости**:
```
fxhash 0.2.1
└── sled 0.34.7 (BoltDB-подобное хранилище)

instant 0.1.13
└── parking_lot 0.11.2
    └── sled 0.34.7
```

**Рекомендации**:
- Рассмотреть миграцию с `sled` на `redb` или `rocksdb`
- Обновить `parking_lot` до версии 0.12+

## 🛡 Безопасность кода

### Положительные аспекты ✅

1. **Хеширование паролей**:
   - Используется `bcrypt` с cost-фактором 12
   - Надёжная защита от brute-force атак

2. **JWT токены**:
   - Алгоритм HS256
   - Время жизни 24 часа
   - Секретный ключ через переменную окружения

3. **SSH ключи**:
   - Поддержка RSA, ED25519, ECDSA
   - Автоматическая установка прав 0o600
   - Поддержка passphrase

4. **Валидация входных данных**:
   - Используется `validator` crate
   - Проверка всех пользовательских данных

### Проблемные области ⚠️

1. **JWT Secret**:
   - По умолчанию используется `"dev-secret-key-change-in-production"`
   - **Требуется**: Обязательная установка через `SEMAPHORE_JWT_SECRET`

2. **RSA уязвимость**:
   - Зависимость `rsa 0.9.10` подвержена Marvin Attack
   - **Требуется**: Мониторинг обновлений

3. **Неподдерживаемые зависимости**:
   - `sled`, `fxhash`, `instant` не поддерживаются
   - **Требуется**: План миграции на альтернативы

## 🔧 Работоспособность проекта

### Статус компиляции

**Результат**: ⚠️ **1334 ошибки компиляции**

**Основные проблемы**:

1. **Конфликты имён** (ambiguous names):
   - Множественные glob импорты в `handlers/mod.rs`
   - Функции с одинаковыми именами в разных модулях

2. **Несоответствие типов**:
   - `max_parallel_tasks: Option<i32>` vs `i32`
   - `ProjectUserRole` vs `String`

3. **Отсутствующие импорты**:
   - `Query`, `Path` из `axum::extract`
   - `Subcommand` из `clap`

4. **Несоответствия имён функций**:
   - `create_project` vs `add_project`
   - `AuthService` vs `LocalAuthService`

### Исправленные проблемы ✅

В ходе проверки исправлено:
- ✅ Удалён дублирующийся файл `projects.rs`
- ✅ Добавлен `once_cell` в зависимости
- ✅ Исправлен тип `BackupProject.max_parallel_tasks`
- ✅ Добавлен `CliResult` тип в CLI модуль
- ✅ Исправлены импорты `LocalAuthService`
- ✅ Добавлены методы `generate_token` и `verify_token`
- ✅ Исправлены импорты `Query` и `Path` в handlers
- ✅ Добавлена функция `extract_token_from_header`
- ✅ Исправлен `version.rs` для обработки отсутствующих переменных окружения
- ✅ Удалён несуществующий импорт `BuildRepo` из git2
- ✅ Исправлены пути к типам (`TotpConfig`, `ProjectUserRole`)

### Требуемые исправления ⚠️
