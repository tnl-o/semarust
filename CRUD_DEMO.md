## 🚀 Быстрый старт

### Вариант 1: Docker (Рекомендуется)

```bash
# 1. Запуск PostgreSQL и инициализация
./semaphore.sh init hybrid

# 2. Запуск сервера
./semaphore.sh start hybrid

# 3. Откройте браузерер
http://localhost/demo-crud.html
```

### Вариант 2: Ручной запуск

```bash
# 1. Запуск PostgreSQL
docker-compose up -d db

# 2. Сборка frontend
cd web
npm install
npm run build

# 3. Запуск backend
cd ../rust
export SEMAPHORE_DB_URL="postgres://semaphore:semaphore_pass@localhost:5432/semaphore"
cargo run -- server --host 0.0.0.0 --port 3000

# 4. Откройте браузер
http://localhost:3000/demo-crud.html
```

## 📖 Что такое CRUD Демо

**CRUD Демо** - это интерактивное демонстрационное приложение, которое показывает все возможности Velum по управлению сущностями через операции:

- **C**reate (Создание)
- **R**ead (Чтение)
- **U**pdate (Обновление)
- **D**elete (Удаление)

### Демонстрируемые сущности

| Сущность | Описание | CRUD |
|----------|----------|------|
| 📁 Проекты | Контейнеры для всех ресурсов | ✅ |
| 📋 Шаблоны | Конфигурация задач автоматизации | ✅ |
| ⚡ Задачи | Запуски шаблонов | ✅ |
| 🖥️ Инвентарь | Список серверов и хостов | ✅ |
| 📦 Репозитории | Git репозитории с playbook | ✅ |
| ⚙️ Окружения | Переменные окружения | ✅ |
| 🔑 Ключи доступа | SSH ключи и пароли | ✅ |
| 🕐 Расписания | Cron-расписания задач | ✅ |

## 📚 Пошаговое руководство

### Шаг 1: Вход в систему

1. Откройте `http://localhost/demo-crud.html`
2. Введите учетные данные:
   - **Имя пользователя:** `admin`
   - **Пароль:** `demo123`
3. Нажмите **Войти**

### Шаг 2: Обзор системы

После входа вы попадете на панель управления:

- **Статистика** - количество всех сущностей
- **Быстрые действия** - создание проекта, шаблона, задачи
- **Навигация** - переключение между разделами

### Шаг 3: Создание проекта

1. Нажмите **"➕ Создать проект"**
2. Заполните форму:
   - **Название:** `My First Project`
   - **Уведомления:** ☑ (опционально)
3. Нажмите **"Сохранить"**

### Шаг 4: Добавление ресурсов

#### 4.1 Инвентарь

1. Перейдите в раздел **"Инвентарь"**
2. Выберите проект из фильтра
3. Нажмите **"Добавить инвентарь"**
4. Заполните:
   - **Название:** `Production Servers`
   - **Тип:** `static`
   - **Данные:** (YAML или INI формат)

```yaml
all:
  children:
    webservers:
      hosts:
        web1.example.com:
          ansible_user: ansible
        web2.example.com:
          ansible_user: ansible
```

#### 4.2 Репозиторий

1. Перейдите в раздел **"Репозитории"**
2. Нажмите **"Добавить репозиторий"**
3. Заполните:
   - **Название:** `My Playbooks`
   - **Git URL:** `https://github.com/myuser/my-playbooks.git`
   - **Ветвь:** `main`

#### 4.3 Окружение

1. Перейдите в раздел **"Окружения"**
2. Нажмите **"Добавить окружение"**
3. Заполните:
   - **Название:** `Production Variables`
   - **JSON:** 

```json
{
  "env": "production",
  "domain": "example.com",
  "ssl_enabled": true
}
```

#### 4.4 Ключи доступа

1. Перейдите в раздел **"Ключи доступа"**
2. Нажмите **"Добавить ключ"**
3. Выберите тип:
   - **SSH Key** - SSH ключ
   - **Login/Password** - Логин/пароль

### Шаг 5: Создание шаблона

1. Перейдите в раздел **"Шаблоны"**
2. Выберите проект
3. Нажмите **"Добавить шаблон"**
4. Заполните:
   - **Название:** `Deploy Web Application`
   - **Playbook:** `deploy-webapp.yml`
   - **Описание:** `Production deployment`
   - **Инвентарь:** выберите из списка
   - **Репозиторий:** выберите из списка
   - **Окружение:** выберите из списка

### Шаг 6: Запуск задачи

1. В списке шаблонов найдите созданный шаблон
2. Нажмите **"▶️ Запустить"**
3. Задача будет создана и выполнена

### Шаг 7: Редактирование сущностей

Для редактирования любой сущности:
1. Найдите сущность в списке
2. Нажмите кнопку **"✏️"**
3. Измените данные
4. Нажмите **"Сохранить"**

### Шаг 8: Удаление сущностей

Для удаления:
1. Найдите сущность в списке
2. Нажмите кнопку **"🗑️"**
3. Подтвердите удаление

⚠️ **Внимание:** Удаление необратимо!

## ❓ Частые вопросы

### Q: Как сбросить демо-данные?

**A:** Пересоздайте контейнер PostgreSQL:

```bash
docker-compose down -v
docker-compose up -d db
```

### Q: Где хранятся данные?

**A:** Данные хранятся в Docker volume `postgres_data`.

### Q: Можно ли использовать демо в продакшене?

**A:** Нет! Демо содержит тестовые данные и упрощенную конфигурацию безопасности.

### Q: Как посмотреть логи backend?

**A:** 

```bash
# Если запускали через start.sh --backend
# Логи будут в терминале

# Или через docker-compose
docker-compose logs -f
```

### Q: Не работает вход в систему

**A:** Убедитесь, что:
1. Backend запущен и доступен
2. PostgreSQL работает
3. Демо-данные загружены

Проверьте:
```bash
docker-compose ps
curl http://localhost:3000/api
```

### Q: Как добавить своего пользователя?

**A:** Через CLI backend:

```bash
cd rust
cargo run -- user add \
  --username myuser \
  --name "My User" \
  --email myuser@example.com \
  --password mypassword123
```
